## Main projects

### MyoGestic

**MyoGestic** is an open-source framework for building real-time biosignal experiments and myocontrol systems.

It is designed to help researchers go from an idea to a working experiment with less boilerplate, including:

- signal acquisition,
- visualization,
- recording,
- model training,
- real-time prediction,
- and closed-loop control.

The goal is to make experimental myocontrol software easier to build, modify, and share.

**Repository:** [NsquaredLab/MyoGestic](https://github.com/NsquaredLab/MyoGestic)

---

### MyoGen

**MyoGen** is an open-source framework for simulating physiologically grounded motor-unit activity, muscle force, and EMG signals.

It is built for researchers who want to:

- simulate motor-unit pools,
- generate synthetic EMG,
- test decomposition or decoding algorithms,
- study how neural drive and muscle properties shape recorded signals,
- and benchmark methods under controlled conditions.

**Repository:** [NsquaredLab/MyoGen](https://github.com/NsquaredLab/MyoGen)

## Why “Myo”?

**Myo** means muscle, and many of my projects are built around muscle signals, motor units, EMG, and the neural control of movement.

But I also like the hidden **“my”** in it.

Because once you clone the repository, adapt it to your experiment, and build something new with it, it becomes **your** tool too.

---

## Open-source philosophy

Scientific code should be:

- readable,
- reproducible,
- inspectable,
- adaptable,
- and shared.

I try to build tools that students, researchers, and developers can open, understand, modify, and use in their own work.
